HOW TO RUN BLOCKVOTE
====================

Two paths: local (Ganache on your machine) or live (Railway-hosted).


LOCAL — five steps
-------------------

Terminal 1: start Ganache with deterministic accounts
    ganache --port 8545 --networkId 1337 --deterministic

Terminal 2:
    cd blockchain && truffle migrate --reset && cd ..
    npm run bundle
    npm run seed        (first time only — seeds admin)
    npm start

Open: http://localhost:8080
Login: admin / admin123
MetaMask: import account 0 private key
    0x4f3edf983ac636a65a842ce7c78d9aa706d3b113bce9c46f30d7d21715b23b1d

Add custom network in MetaMask:
    Name:    Ganache
    RPC URL: http://127.0.0.1:8545
    ChainID: 1337


LIVE (Railway-hosted)
---------------------

Already deployed at:
    App:   https://blockvote-production.up.railway.app
    Chain: https://blockvote-chain-production.up.railway.app

Open the app URL, then in MetaMask:
    Add custom network:
        Name:    BlockVote Chain
        RPC URL: https://blockvote-chain-production.up.railway.app
        ChainID: 1337
    Import account (private key from list in docs/project-explained.md)
    Login: admin / admin123 (when MetaMask is on account 0)


REDEPLOYING THE CONTRACT TO RAILWAY
------------------------------------

In .env (gitignored):
    PRODUCTION_RPC_URL=https://blockvote-chain-production.up.railway.app
    MNEMONIC=myth like bonus scare over problem client lizard pioneer submit female collect

Then:
    npm i -D @truffle/hdwallet-provider                       (one time)
    cd blockchain && truffle migrate --network production --reset && cd ..
    npm run bundle
    git add blockchain/build/contracts/Voting.json
    git commit -m "deploy: new contract address"
    git push origin main

Railway auto-rebuilds the app service in ~2 minutes.


ADMIN WORKFLOW (in browser)
----------------------------

1. Login as admin / admin123
2. Voter Registry — register each voter with voter_id, password, eth address
3. Create Election — name it, sign tx
4. Select the election tab
5. Add candidates (name, party) — one tx each
6. Set Voting Period — start/end dates (UTC midnight is the cut-over)
7. Eligible Voters — click [ whitelist ] for each voter who can vote
   (each click is a tx)


VOTER WORKFLOW (in browser)
----------------------------

1. Login as voter1 / pass1 (or whatever was registered)
2. Switch MetaMask to the address registered for that voter
3. Page shows only elections you're whitelisted for
4. Pick a candidate, click Cast Vote on Blockchain, sign in MetaMask
5. Wait ~2s for confirmation; counts update on reload
